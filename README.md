# Outreach Architect — Lesson 3 Skill

> Скіл для учасників програми **Leadgen & Sales Booster** (Profigent). Архітектор вихідних LinkedIn-продажів на 7 фаз: ICP → підхід → схеми → ПЛАН → пошук Apollo → валідація Apify → персональні дотики.

Канал — тільки LinkedIn, без email.

---

## Що це

**Третій скіл програми.** Будується поверх контекстної моделі з Lesson 1 і контент-двигуна з Lesson 2. Один скіл = повний цикл вихідних продажів в одному діалозі, крок за кроком.

**Залізне правило:** Claude вирішує `КОГО` і `ЧОМУ` → Apollo приносить `СПИСОК` → Apify підтверджує що вони `ЖИВІ` → ти робиш `ДОТИК`.

### Фаза 1 — ICP-інтерв'ю

Питає, кого хочеш у клієнти: індустрія, розмір, гео, посада ЛПР, біль verbatim, buying signal, стек, оффер + головне заперечення. Є контекстна модель з Lesson 1 — підтягне offer/positioning і спитає тільки дірки. Синтез у `outreach/icp.md`.

### Фаза 2 — підхід

Каже, з чого почати саме тобі: з hiring-сигналу, lookalike ідеального клієнта, звуження hard-фільтрами або витягування паттерну з історії розмов.

### Фаза 3 — схеми

Рекомендує 2-3 з 37 схем Claude+Apollo під твою аудиторію + чому НЕ інші. Джерело — `reference/framework-matrix.md`.

### Фаза 4 — план

Показує весь маршрут наперед трьома рядками і чекає твоє `так`: за якими Apollo-фільтрами шукаємо → чим валідуємо в Apify → яким промптом перевіряємо. Це гейт — ніщо не запускається без approve.

### Фаза 5 — пошук Apollo

За погодженими фільтрами дістає приблизно 20 лідів з LinkedIn URL (`apollo:prospect`). Якщо Apollo MCP не підключений — видає готові фільтри JSON-блоком під вставку.

### Фаза 6 — валідація Apify + CSV

Йде в Apify Store, знаходить actor, показує ціну до запуску, пропонує запустити → перевіряє активність, актуальність посади й свіжий тригер → відсіює неактуальних → зберігає `outreach/leads.csv`.

### Фаза 7 — повідомлення

Питає handoff: автоматизація (Tier 2) чи вручну (Tier 1, рекомендується) → під кожного ліда пише коротку персональну LinkedIn-цепочку у твоєму voice: connect без записки → opener → insight → social proof → asset → breakup. Кожне повідомлення <30-35 слів, рівно один CTA, нуль AI-slop.

## Передумова

Перед запуском цього скіла:

1. Запущений скіл Lesson 1 (`company-context-model`) — існує `_context-model.md` з Identity / Offer / ICP / Proof / Voice.
2. Бажано запущений скіл Lesson 2 — існує `voice-profile.json`, щоб цепочка писалась твоїм голосом.
3. Опційно Apollo MCP залогінений (`/mcp` → Apollo → connected) — тоді скіл реально дістає лідів, а не лише радить.
4. Опційно Apify token або скіл `linkedin-enrich` — для валідації аудиторії.

Без Apollo/Apify скіл теж працює в advisory-режимі: він дасть готові фільтри й тексти, а пул ти дістанеш сам.

## Hard rules

1. Спершу план, потім дія. Жоден Apollo-пошук чи Apify-actor не стартує без явного approve на фазі 4.
2. Жоден actor не запускається без показаної ціни.
3. Default = вручну (Tier 1). Скіл драфтить, ти тиснеш send сам.
4. Канал — тільки LinkedIn. Жодних email-послідовностей. Дотики: connect без записки / like / comment / DM.
5. Без `voice-profile.json` цепочка буде generic.
6. Немає відповіді після breakup → стоп. Кожен крок цепочки має власну цінність, не `просто нагадую`.

## FAQ

**Q: Скіл нічого не дістає, тільки радить.**  
A: Apollo MCP не підключений. Підключи (`/mcp` → Apollo → Authenticate) або працюй у режимі порад.

**Q: Цепочка звучить не як я.**  
A: Немає `voice-profile.json`. Згенеруй його в Lesson 2 і поклади в проект.

**Q: Боюся бана за автоматизацію.**  
A: За замовчуванням скіл нічого не відправляє сам. Він драфтить, ти тиснеш send вручну.

**Q: Apify коштує грошей?**  
A: Так, приблизно $1-3 за валідацію 20 профілів. Скіл завжди показує ціну до запуску і чекає твоє `так`.

**Q: Де готовий список лідів?**  
A: Після валідації скіл зберігає `outreach/leads.csv`.

**Q: Чому 2-3 схеми, а не всі 37 одразу?**  
A: Точність важливіша за обсяг. Решту 37 проходь по одній на день з `lesson-3-37-ways-handout.md`.
