# Outreach Architect — Lesson 3 Skill

> Скіл для учасників програми **Leadgen & Sales Booster** (Profigent).
> Архітектор вихідних LinkedIn-продажів на 7 фаз: ICP → підхід → схеми → ПЛАН → пошук Apollo → валідація Apify → персональні дотики.
> _Усе щоб системно знаходити ідеальних клієнтів і торкатися їх у вашому voice. Канал — тільки LinkedIn, без email._

---

## Що це

**Третій скіл програми.** Будується поверх контекстної моделі з Lesson 1 і контент-двигуна з Lesson 2. Один скіл = повний цикл вихідних продажів в одному діалозі, крок за кроком.

**Залізне правило (з лекції):** Claude вирішує `КОГО` і `ЧОМУ` → Apollo приносить `СПИСОК` → Apify підтверджує що вони `ЖИВІ` → ти робиш `ДОТИК`.

### 🧩 Фаза 1 — ICP-ІНТЕРВ'Ю (10 хв)

Питає, кого хочеш у клієнти: індустрія, розмір, гео, посада ЛПР, біль verbatim, buying signal, стек, оффер + головне заперечення. Є контекстна модель з Lesson 1 — підтягне offer/positioning і спитає тільки дірки. Синтез у `outreach/icp.md`.

### 🧭 Фаза 2 — ПІДХІД (2 хв)

Каже, з чого почати саме тобі: з hiring-сигналу / з lookalike ідеального клієнта / зі звуження hard-фільтрами / з витягування паттерну з історії розмов. 2-3 речення маршруту.

### 🎯 Фаза 3 — СХЕМИ (3 хв)

Рекомендує **2-3 з 37 схем** Claude+Apollo під твою аудиторію + чому НЕ інші. Вчить думати, а не копіювати. Джерело — `reference/framework-matrix.md`.

### 📋 Фаза 4 — ПЛАН ⭐ (3 хв)

Показує весь маршрут наперед трьома рядками і **чекає твоє «так»**: *за якими Apollo-фільтрами шукаємо → чим валідуємо в Apify → яким промптом перевіряємо*. Це гейт — ніщо не запускається без approve.

### 🔍 Фаза 5 — ПОШУК `Apollo` (5 хв)

За погодженими фільтрами дістає ~20 лідів з LinkedIn URL (`apollo:prospect`). Якщо Apollo MCP не підключений — видає готові фільтри JSON-блоком під вставку.

### ✅ Фаза 6 — ВАЛІДАЦІЯ `Apify` + CSV (10 хв)

Йде в Apify Store, знаходить actor, показує ціну ДО запуску, пропонує запустити → перевіряє по 3 LinkedIn-вимірах (активність · актуальність посади · свіжий тригер) → відсіює мертвих → зберігає `outreach/leads.csv` (з 20 зазвичай 12-15 живих).

### ✍️ Фаза 7 — ПОВІДОМЛЕННЯ (15 хв)

Питає handoff: 🤖 автоматизація (Tier 2) чи ✍️ вручну (Tier 1, рекомендується) → під **кожного** ліда пише персональну цепочку у твоєму voice: invite → лайк → DM з кейсом → коментар → другий DM → стоп.

## Hard rules (чому деякі речі скіл НЕ робитиме)

1. **Спершу ПЛАН, потім дія.** Фази 1-4 нічого не запускають. Жоден Apollo-пошук чи Apify-actor не стартує без твого явного «approve» на фазі 4.
2. **Жоден actor не запускається без показаної ціни.** Apify коштує грошей — скіл завжди показує ціну ДО запуску і чекає «так».
3. **Default = вручну (Tier 1).** За замовчуванням скіл драфтить, ти тиснеш send сам. Авто-постинг (Tier 2) — окрема історія на свій ризик банy.
4. **Канал — тільки LinkedIn.** Жодних email-послідовностей. Дотики: invite / like / comment / DM.
5. **Без `voice-profile.json` цепочка generic.** Голос підтягується з Lesson 2 — без нього текст звучить як AI, не як ти.
6. **Фінальний DM без відповіді → СТОП.** Не спамити. Кожен крок цепочки має власну цінність, не «просто нагадую».

## FAQ

**Q: Скіл нічого не дістає, тільки радить.**
A: Apollo MCP не підключений. Підключи (`/mcp` → Apollo → Authenticate) або працюй у режимі порад — він дасть готові фільтри, дістанеш сам.

**Q: Цепочка звучить не як я.**
A: Немає `voice-profile.json`. Згенеруй його в Lesson 2 і поклади в проект.

**Q: Боюся бана за автоматизацію.**
A: За замовчуванням скіл нічого не відправляє сам — він драфтить, ти тиснеш send вручну. Ризик ~0. Авто-режим (Tier 2) існує, але це на свій ризик, з лімітом 80-100 інвайтів/тиждень + warm-up.

**Q: Apify коштує грошей?**
A: Так, ~$1-3 за валідацію 20 профілів. Скіл завжди показує ціну ДО запуску і чекає твоє «так».

**Q: Де готовий список лідів?**
A: Після валідації (Фаза 6) скіл зберігає `outreach/leads.csv` — живі ліди + сигнали (active / посада / тригер). Це файл, який ти віддаєш на автоматизацію або береш у роботу вручну.

**Q: А якщо ICP туманний?**
A: Скіл спершу витягне паттерн з історії твоїх розмов (схема #10), а потім будуватиме аудиторію. Або дозаповни `_context-model.md` через `/company-context-model` з Lesson 1.

**Q: Чому 2-3 схеми, а не всі 37 одразу?**
A: Точність > обсяг. Скіл підбирає схеми під саме твою аудиторію і пояснює чому НЕ інші — щоб ти вчився думати, а не копіювати. Решту 37 проходь по одній на день з `lesson-3-37-ways-handout.md`.
